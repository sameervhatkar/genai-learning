1. What is Programming?

Programming is the process of giving instructions to a computer to perform specific tasks using a programming language.

Key ideas:
	•	Computers do not understand human language directly.
	•	They understand structured instructions written in programming languages.
	•	A program is a sequence of instructions executed step-by-step.
	•	Programming helps automate tasks, build applications, and solve real-world problems.

Examples:
	•	Printing output on screen
	•	Performing calculations
	•	Reading files
	•	Building web applications
	•	Training AI models

Python is widely used in:
	•	Backend development
	•	Automation
	•	Data science
	•	Machine learning
	•	Generative AI

5. What is a Virtual Environment (venv)?

A virtual environment is an isolated Python environment created for each project.

It allows:
	•	Separate dependencies per project
	•	Different library versions
	•	Cleaner project structure
	•	Avoid conflicts between projects

Example problem without venv:

Project A needs:

numpy version 1.x

Project B needs:

numpy version 2.x

This causes conflicts globally.

Solution:

Use virtual environments.

Each project maintains its own dependencies.

⸻

6. Creating a Virtual Environment

Command used:

python -m venv venv

Explanation:
	•	python → runs Python interpreter
	•	-m → runs module as script
	•	venv → module name
	•	last venv → folder name for environment

This creates a directory:

venv/

Inside it:
	•	Python interpreter copy
	•	package manager (pip)
	•	environment configuration files

⸻

7. Activating Virtual Environment

Command (Mac/Linux):

source venv/bin/activate

After activation:

Terminal shows:

(venv)

Meaning:

Virtual environment is active.

Now any installed packages remain project-specific.

Example:

pip install requests

Installed only inside this environment.

⸻

8. Deactivating Virtual Environment

Command:

deactivate

Effect:
	•	Leaves virtual environment
	•	Returns to global Python interpreter

⸻

9. Why Virtual Environments Are Important for GenAI Projects

Generative AI projects require many libraries like:
	•	numpy
	•	pandas
	•	transformers
	•	torch
	•	langchain

Different projects may require different versions.

Virtual environments help:
	•	keep projects independent
	•	avoid dependency conflicts
	•	maintain reproducibility
